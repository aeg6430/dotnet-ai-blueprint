# Code Review & Learning Mode

> Modes are configured in `.ai-modes` at the root of the repo.
> Read `.ai-modes` during bootstrap — do not hardcode mode state here.

---

## Code Review Mode

**Controlled by:** `CODE_REVIEW_MODE` in `.ai-modes`

**Trigger phrases** (when mode is ON):
"review this", "check this", "is this good", "what do you think about this code"

Applies to:
- Handwritten C# code
- TODO comments or pseudocode describing intended logic
- Code generated by another AI tool (Copilot, Claude, Cursor, etc.)

**Output format — always in this order:**

1. 🚫 **Problems** — violations of any rule (SOLID, SQL ownership, magic numbers, async pitfalls, coding style, Fowler smells)
2. 💡 **Improvements** — works but could be cleaner, more maintainable, or more idiomatic
3. ✅ **What's good** — explicitly call out what was done well

**Rules:**
- Read fully before commenting — do not interrupt mid-review
- Be specific — name the rule violated, not just "this is bad"
- Review only — do not implement fixes unless explicitly asked

## Learning Mode

**Controlled by:** `LEARNING_MODE` in `.ai-modes`

**When ON:** after every code generation, append a brief explanation:
- **Pattern applied** — what pattern or rule was used and why
- **What was avoided** — what bad pattern was deliberately not used and why
- **Trade-off** — any notable trade-off worth knowing

Keep to 3-5 bullets max. Goal is understanding, not documentation.

**When OFF:** generate silently — no explanations appended.

## Blind Spot Mode

**Controlled by:** `BLIND_SPOT_MODE` in `.ai-modes`

| Value | Behaviour |
|---|---|
| `STRICT` | Stop and ask the user before writing any code when a blind spot is found |
| `WARN` | Flag blind spot as a comment at the top of generated code, proceed with best-guess defaults |
